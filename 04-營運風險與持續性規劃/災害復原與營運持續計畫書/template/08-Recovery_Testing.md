# 捌、系統復原實作測試與紀錄

## 8.1 系統復原測試說明

本章節描述系統復原計畫的測試安排、執行流程與記錄。

### 8.1.1 測試目的

- 驗證復原計畫的可行性
- 驗證 RTO/RPO 目標的達成
- 識別計畫中的不足之處
- 培訓相關人員
- 建立復原操作經驗

### 8.1.2 測試類型

#### 桌上模擬測試 (Tabletop Exercise)

- **頻率**: 每年 1 次
- **時間**: 2-4 小時
- **參與人**: 計畫相關人員
- **內容**: 討論和走過復原步驟
- **成本**: 低
- **中斷業務**: 否

#### 功能測試 (Functional Test)

- **頻率**: 每季 1 次
- **時間**: 4-8 小時
- **參與人**: 技術人員
- **內容**: 執行部分復原步驟，測試特定功能
- **成本**: 中
- **中斷業務**: 否 (測試環境)

#### 完整復原測試 (Full Recovery Test)

- **頻率**: 每年 1-2 次
- **時間**: 1-2 天
- **參與人**: 所有相關人員
- **內容**: 執行完整復原流程，驗證所有功能
- **成本**: 高
- **中斷業務**: 是 (需要安排在非業務關鍵時段)

### 8.1.3 測試排程

| 月份 | 測試類型 | 負責人 |
|------|---------|--------|
| 3 月 | 功能測試 | |

| 12月 | 完整復原測試 + 桌上模擬 | |
